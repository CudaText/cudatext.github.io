Command Line

Executable file "Syn.exe" supports the following command line notations:

  • Syn
  • Syn "file1" [ "file2" ... ] [ options ]
  • Syn "folder1" [ "folder2" ... ] [ options ]
  • Syn "/two=file1|file2|line1|line2|column1|column2|delta"
  • Syn "/cmp=file1|file2"
  • Syn /reg

When run without parameters, program opens empty editor window or the last session/project.
When run with filenames list, program opens given files in separate tabs.
Instead of passing file-names, you can pass folder-names - all text files from specified folders and all their subfolders will be opened.

When run with "/two" key, program shows two files, in two groups, and scrolls both files to specified line/column positions. Line and column numbers are optional. "Delta" means vertical offset between caret and editor edge (minimal is 0).
When run with "/cmp" key, program calls file-compare plugin for two files (only if such plugin installed).
When run with "/reg" key, program only shows "Integration with Explorer" dialog.

Optional keys are:

  • /ro - Set read-only mode for opened file(s).
  • /s - Force option "Allow to run single instance only" to on.
  • /n=Number - Go to specified line number after opening file(s).
