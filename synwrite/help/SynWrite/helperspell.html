Syntax-Aware Spelling

Spell-checking feature in SynWrite is syntax-aware. This means that only those words are spell-checked, which are inside syntax comments and strings. For example, in C language syntax comments are /* Text here */ and syntax strings are double-quoted: "Text here".

How to enable syntax-aware spelling for new lexers?

To use this feature in custom (user-installed) lexers, you need to edit file "Lexers.cfg" in SynWrite installation folder. In this file you need to specify styles of your lexer: comment styles in section [CommentStyles] and string styles in section [StringStyles]. Several styles are comma-separated.

Note that "Lexers.cfg" file is overwritten on each installation. So better write your strings to file "LexersEx.cfg" (create it in the same folder).
