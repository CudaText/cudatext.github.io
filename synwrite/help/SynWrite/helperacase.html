Auto-Correction of Id Case

In the Options dialog, "Lexer Overrides" tab, you can enable auto-correction of identifiers' case.
This feature needs auto-completion file (with extension "acp") for lexer. Such acp-files exist already for most of popular lexers.

How this works: you type valid id but in wrong case, then you type any non-word character, and editor corrects your id. For example, for Pascal lexer: you type "WRIteln(", and editor replaces this to "WriteLn(".

Notes:

  • If lexer allows id with spaces or other spec-chars (e.g. VBScript allows id "End If"), then such id cannot be corrected. You need to have all parts of id (e.g. "End" and "If") listed in acp-file.
  • This feature works only when you're not typing syntax comments/strings. See topic Syntax-Aware Spelling for info about configuring syntax comments and strings.
