Sessions

Sessions are partial states of the program, which are saved into files with "synw-session" extension. You can find sessions control commands in the main menu, under "File -- Sessions". If some session is opened, its name is shown in the SynWrite window caption in curly brackets { }.

Sessions hold the following properties:

  • grouping mode
  • groups splitter position (for modes 2x1, 1x2, 1+2)
  • properties of each named tab:
    • file name
    • group index
    • scroll position (ie. number of top visible line)
    • caret position
    • state "read-only"
    • state "word wrap"
    • state "code folding enabled"
    • state "line numbers visible"
    • mouse selection mode (normal/lines/columns)
    • editors splitter position (each editor can be splitted to master/slave editors)
    • folded ranges list
    • column markers positions
    • tab color

Default session with name "(default)" is used when you didn't create named session but the option "Save last session on exit" is in action. File of this session is saved near "Syn.ini" file (default folder is "%Appdata%\SynWrite").

If you want to open session files by double-clicking on them in Windows Explorer, check out the SynWrite dialog "Options -- Explorer integration".

Command "Open/Add session" does the following: it adds tabs from session to currently opened tabs.
