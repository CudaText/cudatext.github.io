Encoding menu

There are two submenus in "File -- Encoding" menu:

  • Reload as
    This re-reads file from disk in selected encoding.
    If file is no-named, only in-memory encoding is changed.
    If file is named and modified, then additional confirmation allows to choose: re-read file or change encoding in memory only.
  • Convert to
    This only changes encoding in-memory, without file re-reading or saving.
    After that you should save the file.

An example shows difference between "Reload as" and "Convert to":

  • Reloading:
    OEM char #169 "®" → ANSI char #169 "©"
  • Converting:
    OEM char #169 "®" → ANSI char #174 "®"

Also you can click statusbar field with encoding name, this is the same as "Reload as".


How to adjust UTF-8 detection?

If you see some UTF-8 file (without so called BOM signature) loaded in ANSI encoding, this means that UTF-8 detection failed. This is because detection analyzes only first 32Kb of text, so it may fail on bigger files. To adjust the detection buffer size, use hidden option in Syn.ini file: 

[Setup]
Utf8Buffer=128

(The value is buffer size in Kb. This line is absent by default in [Setup] section.)


How to edit encodings menu?

You can edit predefined encodings list (except Unicode list) in the file "Enc.cfg", in SynWrite installation folder. It is text file in INI format. Note that this file is overwritten on each installation.
