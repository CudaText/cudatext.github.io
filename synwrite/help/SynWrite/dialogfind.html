Special characters

If you have checked dialog option "Special characters" (it can't be checked together with "Regular expressions"), the special chars syntax is activated for input fields ("Search for" and "Replace with"). The following special text strings are handled in this mode as a single char:

\n Newline char (same as regex \x0A)
\r Carriage-return char (same as regex \x0D)
\t Tabulation char (same as regex \x09)
\\ Backslash char "\"
\0 Null char (same as regex \x00)

Dialog options

"Options" group

Case sensitive Makes search case-sensitive, i.e. words "Some" and "some" will be different.
Whole words Allows to find only whole words, i.e. word "some" cannot be found inside "awesome" or "somewhat".
Regular expressions Activates RegEx syntax for input text fields ("Search for" and "Replace with").
Special characters Activates special characters syntax for input text fields.
Prompt on replace Enables confirmation messagebox on doing "Replace all" operation, for each found position.
Select results Enables to place multi-selections on found matches, for "Find all" operation.
(When option is off, only search marks will be placed on found matches.)
Bookmark results Enables to place bookmarks on each found line, for "Find all" operation.
(To remove bookmarks, use menu command "Bookmarks -- Clear all bookmarks".)
Extend selection Performs "Extend selection" operation for "Find next" operation. I.e. text will be selected from the edge of previous selection block (or caret position if nothing selected) to the found position.

"Scope" group

Search in selection Performs search only inside selected block.
(Note that search in selection makes new smaller selection, so you can't find all matches in selection by pressing "Find next" button.)
Search from caret Starts search from caret (text blinking cursor) position, not from text beginning or ending.
(Note: option is ignored for "Replace all" operation.)
Wrap search Enables to continue search from the opposite text edge (i.e. continue from text beginning when ending is reached, or continue from text ending when beginning is reached with backward search).
Skip collapsed Disables search inside collapsed text blocks, which are made using "text folding" editor feature.
Entire text/
Comments/
Strings/ ...		 Enables syntax-sensitive search. You can allow to find positions only inside syntax comments (e.g. /*....*/ in C language), or in syntax strings (e.g. double-quoted strings in C language), or in both comments and strings. This way you can minimize number of false positives in search.
(Option has no effect if lexer is not active or lexer does not "know" comments/strings.)

Blue labels

»? Shows menu with quick help on RegEx syntax.
>> Switches dialog between compact and normal modes.
»Search
»Replace		 Switches dialog between Find and Replace modes.
»Multiline Switches dialog between multi-line and single-line views.
Note: this doesn't affect RegEx modifiers, it's only to show multi-line edit fields.
»Docked Switches dialog between "docked into main form" and "separate dialog" modes.
Note: in the docked mode some dialog features don't work (transparency, hotkeys).
Transparency Shows more options: the level of dialog transparency (from "disabled" to "maximal"), and option "on losing focus" (it enables transparency only after switching focus out of this dialog).

Keyboard shortcuts

The following keyboard shortcuts are available in Find/Replace dialogs:

  • F1 - Show this help topic.
  • F2 - Toggle multi-line mode.
  • F4 - Toggle RegEx mode.
  • F5 - Show dialog to enter Unicode hex code to insert.
  • Ctrl+D - Toggle docked mode.
  • Ctrl+F1 - Show quick RegEx help menu.
  • Ctrl+F / Ctrl+H - Toggle Find/Replace mode. (Note: hotkey may be redefined in options.)

When edit field is focused:

  • Alt+Down - Show history popup menu.
  • Ctrl+Down - Copy text from "Search for" to "Replace with" field.
  • Alt+Delete (while combobox is expanded) - Delete selected item from combobox dropdown list.
  • Ctrl+E - Paste from clipboard with regex-escaping (ie, prepend special regex chars with a backslash).

Feature: if Ctrl is pressed, "Replace" action won't jump to next found match after replacing current match (good for very big files).
