Введение

На панель плагина "SynFTP" можно переключиться ярлыком вкладки, внизу панели "Дерево/Проект".

На этой панели можно просматривать FTP или SFTP папки, скачивать файлы, а также автоматически выкладывать их обратно после локального редактирования. Еще можно выкладывать новые файлы на удаленные папки: откройте локальный файл, активируйте SynFTP, перейдите к нужной папке и нажмите кнопку "Upload file".
Этот плагин - портированая версия плагина NppFTP из Notepad++.


Настройки

Есть два диалога для настройки SynFTP. Команды для их вызова можно увидеть, если кликнуть по иконке "шестеренка".

Общие настройки

В диалоге "General settings" можно указать умолчательное место для сохранения файлов. Подробнeе в Пути кэшей. Параметр "Global cache" задает соответствие для корневой папки сервера (т.е. /). Если соответствий в профиле (Profile settings) не назначено, эта папка будет основной папкой кеша.

Настройка профиля

В диалоге "Profile settings" можно создавать/изменять/удалять профили. По умолчанию нет ни одного профиля и соединения невозможны. Для создания нового профиля нужно нажать на Add и ввести для него имя. Переименование и удаление выполняются соседними кнопками.

На вкладке "Connection" задаются параметры для соединения. Минимальный необходимый набор: имя/адрес сервера и порт.
На вкладке "Transfers" можно указать параметры для работы ftp-протокола.
На вкладке "Cache" можно указать соответствия для выбранного профиля. Подробнeе в Пути кэшей.

Приватный ключ

В диалоге профиля можно задать использование "private key file": это гораздо более безопасно, чем пароль. Очистите поле пароля и отметьте опцию "Try private key file authentification". Нужен файл в формате OpenSSH key. Укажите путь к файлу в поле ниже. Поле "Passphrase" нужно для расшифровки файла.

Скрытые опции

Некоторые опции SynFTP можно настраивать только через редактирование файла "SynFTP.xml", расположенного по умолчанию в папке "%AppData%\SynWrite\SynFTP".

  • опция "menuSplit" задает максимальное число строк в выпадающем меню профилей. Например, если задано 20, то меню профилей будет иметь 20 строк в одной колонке.

Пути кэшей

Когда файлы скачиваются с сервера, они сохраняются в кэше. Когда файл обновляется в кэше, он автоматически копируется на сервер. Для обеспечения более полного контроля над расположением и передачами файлов можно создать сопоставление локальных и удаленных папок. "Local path" будет указывать место для на локальном компьютере, где будут искаться файлы для копирования на сервер. Например, если указать"C:\ftpfiles\myserver\home", то файлы из этой папки и ее подпапок будут передаваться в соответствующие папки на сервере. "External path" указывает место на сервере, например "/home/myuser/public_html/". Из нескольких вариантов сопоставлений выбирается первый подходящий (даже если есть "более подходящий"). Например, рассмотрим такой сценарий:

Сопоставление в профиле: 

Local                        External
C:\webfiles                  /home/user/public_html
C:\webfiles                  /home/user2/public_html
C:\rootfiles                 /root
D:\serverfilesystem          /

Сопоставление в диалоге "Global settings": 

C:\myuser@server.com\        / (не меняется)

Скачивание: 

Внешний файл "/home/user/public_html/index.html" скачается в "C:\webfiles\index.html"
Внешний файл "/home/user/.bash_rc" скачается в "D:\serverfilesystem\home\user\.bash_rc"
Внешний файл "/root/apache.conf" скачается в "C:\rootfiles\apache.conf"
Внешний файл "/vmlinuz.img" скачается в "D:\serverfilesystem\vmlinuz.img"
Никакие файлы не скачаются в "C:\myuser@server.com\"

Копирование на сервер: 

Локальный файл "C:\webfiles\home\user\.bash_rc" закачается в "/home/user/public_html/home/user/.bash_rc" ("user2" не будет использовано)
Локальный файл "D:\serverfilesystem\boot\grub\menu.lst" закачается в "/boot/grub/menu.lst"
Локальный файл "C:\myuser@server.com\home\user\public_html\index.html" закачается в "/home/user/public_html/index.html"

Итого. Последовательность пар важна. Сопоставление из диалога "Global settings" всегда используется последним, просмотр пар из сопоставления в профиле идет сверху вниз. Так что если пара 

D:\serverfilesystem          /

будет расположена первой, то ВСЕ файлы будут скачиваться в папку "D:\serverfilesystem".


Панель инструментов

На панели инструментов есть следующие кнопки:

  • Connect/Disconnect: Либо подключиться к серверу из указанного профиля (выпадает список профилей), либо отключиться от текущего сервера.
  • Open directory: Перейти к папке сервера по введенному в диалоге пути (пример: "/pub/music").
  • Download file: Если есть выделенный файл в дереве, скачать его в кэш и открыть.
  • Upload file: Если есть выделенная папка в дереве, закачать текущий редактируемый файл в эту папку.
  • Refresh: Если есть выделенная папка в дереве, перечитать ее содержимое.
  • Abort: Если выполняется передача файла, прервать ее.
  • Settings: Вызвать один из диалогов настроек.
  • Show/Hide log panel: Показать/спрятать панель с сообщениями плагина.
  • About plugin: Показать информацию о версии плагина.

Панели

Дерево

Если установлено соединение, дерево показывает файлы на сервере. Некоторые команды из тулбара зависят от текущего выбранного объекта в этом дереве. Двойной клик на папке приводит к отображению ее подъобектов. Двойной клик на файле приводит к скачиванию его в кэш и открытию.

Очередь

Очередь показывает текущую и запланированные передачи файлов, в том числе процент выполнения и путь. Клик правой кнопкой мыши над отдельным элементом позволяет вызвать команды для его обрыва или отмены (в зависимости от того, идет передача или запланированна).

Журнал сообщений

В журнале отображается протокол выполнения операций. Если что-то выполняется неправильно, здесь нужно искать описания ошибок. Оповещения выделены синим, сообщения сервера - зеленым, ошибки - красным. Панель журнала есть часть панели "Вывод" редактора. Она появляется/прячется кнопкой "Show log panel".
