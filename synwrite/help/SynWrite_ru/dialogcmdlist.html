Введение

Эти два диалога функционируют одинаково. В обоих случаях отображается список текстовых элементов и поле для условия поиска. Можно перемещать выделение в списке (даже при фокусе на условии поиска) сочетаниями Up, Down, PageUp, PageDown, Ctrl+Home, Ctrl+End. Если условие поиска не пусто, то отображаются только подходящие ему элементы.


Обычный и нечеткий поиск

Если включена опция "Нечеткий поиск" (внизу диалога), то применяется "fuzzy" алгоритм: подходящими будут считаться те элементы, в которых есть все указанные в условии символы, но не обязательно подряд. Примеры:

  • для "fsav" подходит "File: Save file"
  • для "fsav" подходит "Focusing: Toggle master/slave focus"

Если опция "Нечеткий поиск" выключена, то подходящими будут считаться элементы, содержащие каждое из введенных слов. Примеры:

  • для "fi re" подходит "File: Reopen file"
  • для "fi re ti" подходит "Run: Tidy - configure"

Файлы проекта: имена файлов и папок

Только в диалоге "Файлы проекта":
Можно в условии поиска применять символ "\", чтобы указать на поиск папки и/или файла. Примеры:

  • (обычный поиск) для "mypro\mycla" подходит "D:\Test\MyProject\myclass.py"
  • (обычный поиск) для "\my" или "my" подходят файлы с "My" в имени (но не в пути)
  • (обычный поиск) для "my\" подходят файлы с "My" в пути (но не в имени)
  • (нечеткий поиск) для "te\my\my.py" подходит "D:\Test\MyProject\myclass.py"

Файлы проекта: поиск по содержимому

Только в диалоге "Файлы проекта":
В конце условия на имена файлов можно использовать конструкцию @текст, чтобы оставить только те файлы, которые содержат такую строку текст.

Поиск внутри файлов будет с учетом регистра, если в конструкции использован один "@", и без учета регистра, если применено "@@". Поиск всегда будет внутри слов (не "слова целиком").
