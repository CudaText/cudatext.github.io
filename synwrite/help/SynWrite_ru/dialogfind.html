Специальные символы

При включенной опции "Специальные символы" (она не может бы включена одновременно с "Регулярные выражения") в условии поиска можно применять следующие специальные конструции:

\n Символ новой строки (совпадает с \x0A из рег.выр.)
\r Символ возврата каретки (совпадает с \x0D из рег.выр.)
\t Символ табуляции (совпадает с \x09 из рег.выр.)
\\ Обратная косая черта
\0 Нулевой символ (совпадает с \x00 из рег.выр.)

Настроки диалога

Группа "Опции"

Учитывать регистр Делает поиск регистрозависимым, например, "Some" и "some" будут различаться.
Слова целиком Поиск только целых слов, например, строка "some" не будет найдена внутри "awesome" или "somewhat".
Регулярные выражения Включает поддержку синтаксиса Рег. выражений в полях "Искать" и "Заменить на".
Специальные символы Включает поддержку синтаксиса Спец. символов. (Не работает вместе с рег-выражениями.)
Запрос при замене Включает запрос подтверждения для каждой замены.
Выделить результаты Включает режим, при котором команда "Найти все/пометить" помещает мульти-выделения на все найденные фрагменты. (Если выкл., на фрагменты будут помещены "пометки поиска".)
Пометить закладками Включает режим, при котором команда "Найти все/пометить" расставляет закладки на строки с результатами поиска. (Для удаления этих закладок есть команда меню "Закладки -- Очистить закладки".)
Расширить выделение По команде "Найди далее" предыдущее выделение расширяется так, чтобы включить в него найденный фрагмент. Если выделения не было, то началом будет положение каретки.

Группа "Диапазон"

Искать в выделении Выполнять поиск только внутри выделенного блока.
(Внимание! Первый уcпешный поиск изменит выделение на найденный фрагмент, а значит следующая команда "Найти далее" не сработает.)
Искать от каретки Начинать поиск от позиции каретки (мигающей черточки), а не с начала/конца текста.
Искать по кругу Продолжать поиск с противоположной границы текста (например, с начала при достижении конца).
Пропускать свернутое Не вести поиск внутри свернутых частей текста, которые появляются в результате сворачивания кода (folding).
"Весь текст"/
"Комментарии"/
"Строки"/ ...		 Сделать поиск чувствительным к синтаксическим конструкциям. Можно разрешить поиск только внутри комментариев (например, /*....*/ в языках C/C++/Java/JavaScript/...) или в строковых константах (например, внутри "....") или и там, и там одновременно. Это позволяет минимизировать число ненужных результатов при поиске.
(Эта опция игнорируется, если для текста на задан лексер или в лексере нет комментариев и строковых констант.)

Синие метки

»? Вызов меню с подсказками по составлению регулярных выражений.
>> Переключение диалога в компактный режим.
»Поиск
»Замена		 Переключение между диалогами Поиск и Замена.
»Многострочный Переключение между видами полей ввода текста: однострочным и многострочным.
Замечание: это не влияет на поведение регулярных выражений, а только изменяет поля для ввода условий.
»Встроенно Переключение между режимами диалога: "встроенный в главную форму редактора" и "отдельный диалог".
Замечание: во встроенном режиме некоторые возможности диалога не работают (полупрозрачность, горячие клавиши).
Прозрачность Настройка полупрозрачности самого диалога, а также опция "потеря фокуса", которая указывает, что полупрозрачность нужна только когда диалог не активен.

Клавиатурные сочетания

Доступны следующие сочетания:

  • F1 - Показать эту страницу Справки.
  • F2 - Переключить однострочный/многострочный режим.
  • F4 - Переключить опцию "Регулярные выражения".
  • F5 - Показать диалог ввода Unicode hex-кода для вставки символа.
  • Ctrl+D - Переключить встроенность диалога.
  • Ctrl+F1 - Показать меню с подсказками по составлению регулярных выражений.
  • Ctrl+F / Ctrl+H - Переключить между диалогами Поиск и Замена. (Сочетание меняется в диалоге опций.)

Если активны поля с условиями поиска или замены:

  • Alt+Down - Показать историю полей ввода.
  • Ctrl+Down - Заполнить поле "Заменить на" значением из поля "Искать".
  • Alt+Delete (при раскрытой истории значений) - Удалить из списка выбранный элемент истории.
  • Ctrl+E - Вставить из буфера с экранированием вставленной строки (обратными косыми чертами).

Если зажата Ctrl, команда "Заменить" не будет искать следующее вхождение строки после замены текущего вхождения (полезно для больших файлов).
