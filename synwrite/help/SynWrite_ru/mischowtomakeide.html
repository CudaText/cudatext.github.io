Как создать IDE для вашего языка?

Здесь описывается, как создать свою переносимую сборку SynWrite, которая поддерживала бы работу с нужным языком программирования. Например, уже есть готовая сборка, названная GCB@Syn, для работы с языком GCBasic.

  • Распакуйте архив "SynWrite.rar" (его можно взять с домашней страницы) в новую папку, скажем, "C:\SynWrite".
    Сделайте сборку переносимой, добавив в эту папку пустой файл "Portable.ini".

  • Удалите лишние файлы (если они не потребуются):
    • "Syn2.wlx" (плагин для Total Commander).
    • Папку "Dictionaries", если не потребуется проверка правописания.
    • Лишние языковые файлы из папки "Lang".
    • Возможно, папку "Plugins", а вместе с ней файл "SynPlugins.sample.ini".
    • Все файлы из папки "Readme" кроме "SynWrite.chm".
    • Всю папку "Data", либо отдельные шаблоны из нее:
      • "clips" - папка с текстовыми фрагментами (см. команду "Окна -- Панели -- Панель текстовых фрагментов"),
      • "colors" - папка с наборами цветов для интерфейса SynWrite (см. диалог "Опции -- Цвета"),
      • "newdoc" - папка для команды "Файл -- Создать",
      • и т.д.
    • Возможно, большинство элементов из папки "Tools" (например, ColorPicker.exe, Tidy.exe, Emmet.js и т.д.) или всю папку "Tools".
    • Файл "Tidy.cfg", если не планируется использовать команды из "HTML Tidy".
    • Файл "Pluginst.inf".

  • Запустите SynWrite, откройте диалог "Библиотека лексеров" ("Опции -- Изменение библиотеки лексеров") и удалите лишние лексеры. Сохраните библиотеку. Файл "LexLib.lxl" станет значительно меньше.
    Еще удалите лишние ACP-файлы из папки "Data\autocomplete".

  • Импортируйте нужные вам лексеры (например, "GreatCowBasic" for GCB@Syn) в библиотеку лексеров.
    Положите соответствующие ACP-файлы в папку "Data\autocomplete".

  • Полезно спрятать лишние (например, ориентированные на HTML) пункты меню. Для этого есть команда "Опции -- Расширенные -- Спрятать пункты меню...".

  • Откройте диалог "Внешние инструменты" (см меню "Опции") и создайте в нем подходящие вам элементы для компиляции, запуска BAT-файлов, вызова справки и т.д.
    • Весьма полезно настроить распознавание сообщений компилятора, чтобы можно было от сообщения об ошибке автоматически переходить к строке кода.
    • Полезно назначить клавише F1 вызов элемента со справкой по вашему языку.

  • Не забудьте удалить лишние INI-файлы из папки "C:\SynWrite" перед ее упаковкой.
    Оставьте Syn.ini (здесь сохраняются конфигурации инструментов и опции).
    Удалите SynState.ini, SynStyles.ini и все SynSpell*.

Это практически все. В сборку GCB@Syn еще добавлен инструмент IDE.exe, который устанавливает переменные окружения для GCB, так что внешние инструменты могут полагаться на эти переменные (IDE.exe запускает Syn.exe). Это может потребоваться, если придется вызывать приложения не из папки SynWrite.
